# 品牌／公眾人物避雷查詢

這是一個純靜態的避雷查詢頁，用來查詢使用者設定的個人避買品牌、公眾人物、政黨人物名單、旅遊目的地風險提醒，以及公開來源可支持的中共政權關聯風險廠商或政治表態風險項目。

線上版：

```text
https://ml0427.github.io/brand-avoidance-query/
```

頁面讀取同資料夾的 `brands.json`，提供品牌／人物／政黨名單搜尋、可信度篩選、狀態篩選、大分類篩選、避買理由篩選與資料來源顯示。

## 使用方式

在搜尋框輸入品牌名、公司名、公眾人物姓名、旅遊目的地、常見別名或辨識編號即可查詢，例如：

- `TikTok`
- `MG4`
- `DJI Mini`
- `味全`
- `83035103`
- `成龍`
- `統促黨`
- `王耀慶`
- `黃國昌`
- `國民黨`
- `柬埔寨`
- `韓國旅遊風險`

搜尋框只查名稱、別名與辨識編號，不查一般品項文字或泛稱。像 `珍珠芭樂`、`鳳梨酥`、`行動電源` 這種沒有品牌限定的普通商品名，或 `藝人`、`政治人物` 這種泛稱，不會被當成命中。

## 本機開啟

請用本機 HTTP 靜態伺服器開啟，不要直接雙擊 `index.html`。

PowerShell：

```powershell
.\start-server.ps1
```

開啟：

```text
http://localhost:4173/
```

若 4173 已被使用：

```powershell
.\start-server.ps1 -Port 4174
```

## 資料

`brands.json` 目前有 158 筆資料：

- 使用者指定的個人避買／辨識字樣。
- 公開來源可支持的中共政權關聯風險廠商。
- 公開來源可支持的公眾人物政治表態、統戰組織或中共官方互動風險。
- 使用者指定的政黨人物名單與長期中國大陸市場發展藝人名單；這類資料用 `personal` 標示，不等同中共關聯或違法定論。
- 使用者指定的旅遊目的地／司法安全風險提醒；這類資料用 `personal` 或 `watchlist` 標示，不取代外交部、陸委會或所在地官方即時公告。
- 使用者指定的社群政治爭議／體育協會預算治理線索；這類資料用 `watchlist` 標示，只記錄可查證爭議邊界，不等同個別人物、協會違法或貪污定論。
- 依證據強度標示 `high`、`medium`、`low`，狀態則區分 `來源確認`、`觀察中`、`個人避買`。

收錄門檻不是「中國製造」，而是至少有以下公開來源之一：

- 國企、央企或地方國資控制。
- 公開黨組織、黨建治理或黨企融合資料。
- 政府制裁、出口管制、國安清單或政府資安警示。
- 人權監控、強迫勞動或供應鏈透明度風險來源。
- 使用者明確設定的個人避買資料。
- 公眾人物或政治組織的公開促統、一國兩制、港區國安法、撐警、統戰組織或中共官方政治職務來源。
- 使用者指定納入的政黨現任人物名單，或長期中國大陸市場發展藝人名單；這類不自動代表親共、違法或受制裁。
- 使用者指定的旅遊目的地風險分級；此類不代表官方旅遊禁令，也不代表所有旅客必然遭遇相同風險。
- 使用者指定的社群政治爭議或協會治理線索；此類需保留「可確認事實／未確認指控」邊界，不寫成個別人物或組織違法定論。

低信心或只有調查性質的項目應放 `watchlist`，不要寫成法律定罪。此清單是避買查詢工具，不是法律裁判，也不代表台灣禁售或證明每項商品都有問題。

## 資料欄位原則

- `aliases` 只放品牌名、公司名、明確系列名或常見通路名稱。
- 公眾人物只放本名、藝名、常用外文名或正式組織名。
- 沒有品牌限定的普通商品名，例如水果品種、鳳梨酥、熱水壺、行動電源，不放 `aliases`。
- `藝人`、`政治人物`、`主持人`、`主席`、`候選人`、`歌手`、`演員` 等泛稱或職稱，不放 `aliases`。
- 政黨名單可以放政黨名與可核對姓名，但不把一般黨員、職稱、地方議員或候選人泛稱硬塞成 alias。
- 旅遊目的地風險可以放國家／地區名與明確風險查詢語，例如 `柬埔寨旅遊風險`；不把 `旅遊`、`旅行`、`出國`、`國家`、`機票` 等泛稱作有效命中。
- 統一編號等數字放 `identifiers`，顯示為「辨識編號」，不混在別名。
- 搜尋框只查品牌名、別名與辨識編號；分類與避買理由用下拉篩選。
- `categories` 使用大類，不放細項。

現有分類固定為：

- 食品／飲料／農水產
- 科技／通訊／消費電子
- 監控／AI／資安
- 電商／App／數位服務
- 家電／智慧家庭
- 汽車／交通／能源
- 服飾／玩具／生活零售
- 餐飲／生活服務
- 媒體／娛樂／公眾人物
- 政治／統戰／公眾人物
- 政治／政黨人物名單
- 體育／協會治理

## 更新資料

若要重跑本輪親共／中共政權關聯風險資料合併，來源資料位於 `data/records/*.mjs`，共用 helper 與清理規則位於 `data/record-helpers.mjs`、`data/record-overrides.mjs`：

```powershell
node .\scripts\merge-risk-records.mjs
```

重跑後執行固定驗證腳本：

```powershell
node .\scripts\validate-brand-records.mjs
```

需要針對新紀錄做正查／反查時，可加上 target 與查詢矩陣：

```powershell
node .\scripts\validate-brand-records.mjs --target-id <record-id> --positive <品牌名> --positive <公司名> --negative <泛稱>
```

這個腳本會自動執行：

- `brands.json` JSON parse、normalize 與 `validateBrandRecord()`。
- duplicate ID 檢查。
- README 筆數一致性檢查。
- 預設正查 smoke tests。
- 預設泛稱反查，避免 `牙膏`、`水`、`旅遊`、`政治人物` 等普通詞誤命中。
- `node --check app-core.mjs`、`node --check app.js`、`node --check scripts\*.mjs`、`node --check data\**\*.mjs`。
- `git diff --check`。
- 發布 gate 必須是 0 errors、0 warnings；`--allow-warnings` 只用於調查，不可作為自動發布依據。

正式交辦的入庫／更新任務，在驗證腳本 PASS（0 errors、0 warnings）、diff 只包含預期檔案、本地 AI 筆記已更新後，可以直接 commit 並 push 到 `origin/main`。Pinecone 對此專案預設視為淘汰／非必要，不作例行同步；除非老爺明確要求或 legacy 查找仍依賴它，否則以本機檔案、Obsidian、AI notes 與 git history 作為耐久紀錄。主動探索候選仍不可自動入庫或自動發布，必須等使用者確認候選。
